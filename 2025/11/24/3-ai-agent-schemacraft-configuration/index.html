AI Agent SchemaCraft 配置指南

概述

SchemaCraft 是 UniCraft 低代码平台的核心 AI 组件，负责将自然语言业务需求转换为符合规范的 Vue 组件 SchemaNode 配置。本指南详细介绍如何在 HiAgent 平台配置和部署 SchemaCraft Agent。

核心架构

Agent 功能定位

SchemaCraft Agent 作为低代码平台的”翻译器”，将用户的自然语言描述转换为结构化的组件配置：

  • 输入: 自然语言业务需求（如”创建一个用户注册表单”）
  • 输出: 符合 SchemaNode 规范的 JSON 配置
  • 核心能力: 组件选择、属性配置、数据绑定、事件处理

阶段 3: Prompt 工程配置

3.1 System Prompt 配置

System Prompt 是 Agent 的核心，包含以下关键部分：

角色定义:

关键约束（6项必须遵守）:

  1. 组件白名单约束: 所有 component 值必须在允许列表中
  2. 数据路径格式: 使用 ${namespace.path} 表达式格式
  3. 命名空间限制: 禁止使用 window、document、globalThis
  4. JSON格式要求: 纯 JSON 格式，使用双引号，无尾随逗号
  5. 必需字段: 每个 SchemaNode 必须包含 component 字段
  6. 事件配置格式: events 字段值为字符串处理器名称

组件库支持:

  • Element UI: el-button, el-input, el-form, el-table 等
  • MS UI: ms-table, ms-form, ms-page-header 等
  • HTML 标签: div, span, p, h1-h6, a, img 等

错误处理策略

降级机制:

  • AI 生成失败 → 规则引擎兜底
  • API 异常 → 缓存结果返回
  • 超时处理 → 返回标准模板

告警配置:

  • 错误率 > 10% 触发告警
  • 响应时间 > 30秒 触发告警
  • API 调用失败触发告警

常见问题解决

问题 1: 输出格式错误

现象: Agent 返回的 JSON 格式不正确或无法解析

解决方案:

  1. 检查 Temperature 设置 (推荐 0.2-0.3)
  2. 加强 System Prompt 中的格式约束
  3. 启用严格 JSON 验证
  4. 增加输出格式示例

问题 2: 组件选择错误

现象: 使用了不在白名单中的组件

解决方案:

  1. 更新组件库白名单说明
  2. 增加组件选择示例
  3. 上传最新的组件库文档

问题 3: 响应时间过长

现象: Agent 响应时间超过预期

解决方案:

  1. 减少 Few-Shot 示例数量 (3-5个)
  2. 优化 Prompt 长度
  3. 调整 Max Tokens 设置
  4. 启用响应缓存

问题 4: API 调用失败

现象: 无法调用 Agent API 或返回错误

解决方案:

  1. 检查 API 端点 URL 和密钥
  2. 验证网络连接状态
  3. 确认请求格式符合规范
  4. 查看 HiAgent 平台错误日志

总结

SchemaCraft Agent 的成功配置需要关注以下关键点：

  1. 精确的约束定义: 通过详细的 System Prompt 确保输出符合规范
  2. 丰富的示例支持: Few-Shot Learning 提供学习样本
  3. 完整的知识库: 规范文档和示例库支持复杂查询
  4. 严格的质量控制: 多层次验证确保输出质量
  5. 持续的优化改进: 基于监控数据和用户反馈持续优化

通过合理的配置和持续的优化，SchemaCraft 能够稳定地将自然语言需求转换为高质量的 Vue 组件配置，为低代码平台提供强大的 AI 辅助功能。
