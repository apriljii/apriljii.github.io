对象 继承 多态

前言

在上一篇文章中，我们介绍了 JavaScript 面向对象编程的基础知识，包括对象、构造函数、原型系统等核心概念。本文将继续深入探讨面向对象编程的两个重要特性：继承（Inheritance）和多态（Polymorphism）。这些概念是面向对象编程的精髓，能够帮助我们更好地组织和复用代码。

继承（Inheritance）

继承是面向对象编程的一个重要特性，它允许我们创建一个新类（子类）来继承另一个类（父类）的属性和方法。子类可以拥有父类的所有特性，同时也可以添加自己的特性或修改父类的行为。

1. JavaScript 中的继承

JavaScript 通过原型链实现继承，这与其他基于类的语言（如 Java、C++）不同。JavaScript 的继承是基于原型的委托机制。

2. 继承的实现方式

2.1 原型链继承

优点： 简单，实例可继承父类的属性和方法
缺点： 1. 无法向父类构造函数传参 2. 所有实例共享引用类型的属性

2.2 构造函数继承

优点： 1. 可以向父类传参 2. 避免引用类型属性共享
缺点： 无法继承父类原型上的方法

2.3 组合继承（推荐）

优点： 结合了原型链继承和构造函数继承的优点
缺点： 会调用两次父类构造函数

多态（Polymorphism）

多态是指同一个方法在不同对象上具有不同的行为。多态允许我们用统一的方式处理不同类型的对象。

1. 多态的概念

2. 多态的好处

  1. 代码复用性：使用统一的接口处理不同类型的对象
  2. 可扩展性：添加新的子类不需要修改现有代码
  3. 灵活性：运行时决定调用哪个方法

面向对象设计的原则

1. 单一职责原则（SRP）

一个类应该只有一个引起它变化的原因。每个类应该专注于做一件事情。

2. 开闭原则（OCP）

软件实体（类、模块、函数等）应该对扩展开放，对修改关闭。

3. 里氏替换原则（LSP）

子类对象应该能够替换父类对象使用，而不破坏程序的正确性。

4. 依赖倒置原则（DIP）

高层模块不应该依赖低层模块，两者都应该依赖抽象。

5. 接口隔离原则（ISP）

客户端不应该依赖它不需要的接口。

6. 迪米特法则（LoD）

一个对象应该对其他对象有最少的了解。
