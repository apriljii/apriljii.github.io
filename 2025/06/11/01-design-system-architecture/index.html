MS UI Design System - 1. 设计规范体系与分层架构

April Ji
2025-06-11
Vue.js 组件库 前端架构 金融科技 Element UI 设计系统

本文将深入探讨 MS UI Design System 的设计规范体系，重点介绍设计分层架构、视觉语言系统，以及如何在金融科技场景下构建完整的设计规范。

前言

在现代前端开发中，设计规范不仅是设计师的产出，更是开发团队的行动指南。MS UI Design System 基于 Element UI 构建了一套完整的设计规范体系，通过科学的设计分层架构，确保了设计规范在不同层面的统一性和可维护性。

本文将从设计分层架构入手，详细讲解 MS UI Design System 的设计规范实现。

一、设计分层架构：规范的科学组织方式

1.1 设计分层的核心理念

MS UI Design System 的设计分层架构基于这样的理念：设计规范不应是平面化的规则集合，而应该是层次化的系统架构。

1.2 整体交互设计规范的构成

除了上述 Foundation/Component/Application 的技术分层外，在协作流程上，我们还定义了明确的规范分层，以确保各角色的产出对齐：

  1. 基础规范（Tokens/标准）：色彩、间距、字体等是当前统一标准，也是裁决依据。出现不一致，先回到这一层对齐。
  2. 设计稿（视觉表达层）：设计稿负责把标准“画出来”，体现布局、视觉层级、组件视觉样式和页面模式，帮助我们做视觉对齐。过渡期如果与基础规范冲突，以基础规范为准。
  3. 交互指南（交互规则层）：负责把状态、反馈、防呆、加载/空态等高频争议点固化为统一规则，保证“用起来一致”。遇到交互分歧，按交互指南裁决。
  4. 页面级示例（Demo/骨架层）：我们暂时提供了两个搜索列表页 Demo（顶栏方案、侧边栏方案），它们是可复用的页面骨架：导航怎么组织、搜索怎么放、内容区怎么分组、状态与反馈怎么处理。目标是让大家直观的看到规范后的 web 页面示例。

六、总结

MS UI Design System 的设计分层架构为金融科技场景提供了一套完整、可维护的设计规范体系：

  1. Foundation Layer 提供了基础的设计 Token，确保了设计的原子化
  2. Component Layer 将 Token 转换为具体的组件规范，保证了组件间的一致性
  3. Application Layer 处理主题和上下文适配，满足不同场景的需求

通过这种分层架构，MS UI Design System 不仅继承了 Element UI 的成熟设计，还在金融科技场景下进行了针对性的优化，确保了设计规范的实用性和可维护性。
