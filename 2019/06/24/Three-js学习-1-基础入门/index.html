Three.js学习(1)-基础入门

April Ji
2019-06-24
Three.js

前言

最近想学习 Three.js，想着做个炫酷效果，结果发现最基础的东西都还没搞清楚。Three.js 是个基于 WebGL 的 JavaScript 库，让我们在浏览器里搞 3D 图形不用直接面对一堆复杂的 WebGL API。

本系列笔记想把学习过程记录下来，从最基础的开始慢慢深入。希望能帮到跟我一样刚入门的小伙伴们。

Three.js 到底是啥

简单说，Three.js 就是帮我们封装好了 WebGL 的那些复杂操作。你知道 WebGL 吧？就是浏览器里的 3D 绘图标准，但直接用它写代码简直要命，一堆数学计算不说，还要处理各种底层细节。

Three.js 就好比在 WebGL 外面包了一层棉花糖，让我们用更简单的 API 就能做出 3D 效果。核心就三个东西：场景、相机、渲染器。搞定了这些，就能开始画了。

场景(Scene)

场景就像个大容器，把要显示的所有 3D 对象都放进去。它本身是看不见的，就负责管理，管着里面所有的东西。

相机(Camera)

相机决定了你从哪个角度看场景。就像拍照的时候，可以选近景、远景、广角。Three.js 里最常用的是透视相机，看起来更真实。

渲染器(Renderer)

渲染器就是把场景和相机结合，生成最终的画面显示在屏幕上。WebGLRenderer 最常用的那个。

写在最后

折腾了半天，终于把 Three.js 的基础搞明白了。总结一下学到了啥：

  1. 三大件 - 场景、相机、渲染器，缺一不可
  2. 形状+材质 - 几何体加材料才是个完整的物体
  3. 光线很重要 - 上帝说，要有光
  4. 动起来才好玩 - 动画让静态的 3D 变活了

现在回头看第二篇的地球绘制，应该会轻松很多。毕竟基础打好了，后面就是各种高级技巧的叠加。下次来画个会转的地球！
